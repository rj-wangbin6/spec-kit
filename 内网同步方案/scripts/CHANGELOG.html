v0.0.77 最新版本
📅 2026-04-09

🎯 本次更新重点

修复 PowerShell 脚本执行策略错误，扩展多 Python 版本支持（3.11、3.12、3.13）。

🐛 Bug 修复

  • 修复 PowerShell 执行策略错误: 离线模式调用 PowerShell 脚本时添加 -ExecutionPolicy Bypass 参数，解决 Windows 执行策略限制导致的脚本执行失败
    现象： 运行 specify init --ai copilot --offline 时报错 "因为在此系统上禁止运行脚本 SecurityError: UnauthorizedAccess"

    根因： Windows 默认执行策略可能设置为 Restricted，阻止未签名脚本执行

    修复： 添加 -ExecutionPolicy Bypass 参数到 PowerShell 命令调用
    📁 src/specify_cli/__init__.py#L1375

✨ 功能改进

  • 支持多 Python 版本: 内网服务器现在提供 Python 3.11、3.12、3.13 的依赖包，用户可以在任意支持的 Python 版本上安装 specify-cli
  • 更新同步脚本: sync-spec-kit.sh 改为循环下载三个 Python 版本的依赖，确保多版本兼容性
    📁 内网同步方案/scripts/sync-spec-kit.sh

📦 部署信息

  • 内网服务器版本: 0.0.77
  • 支持的 Python 版本: 3.11, 3.12, 3.13

v0.0.76
📅 2026-04-09

🎯 本次更新重点

统一离线和非离线安装的目录结构，确保所有安装方式都包含 .claude/skills/ 目录。

🐛 Bug 修复

  • 修复离线安装缺少 .claude/skills/: 离线安装现在也会将 skills 移动到 .claude/skills/ 目录，与非离线安装保持一致
    现象：
    • specify init --here --ai copilot (非离线) → 有 .claude/skills/ ✅
    • specify init --here --ai copilot --offline (离线) → 缺少 .claude/skills/ ❌

    根因： 非离线安装调用了 move_skills_to_claude()，但离线安装的 scaffold_from_core_pack() 未调用此逻辑

    修复： 在 scaffold_from_core_pack() 末尾添加 skills 移动逻辑
    📁 src/specify_cli/__init__.py#L1425

✨ 功能改进

  • 目录结构统一: 两种安装方式现在生成完全一致的目录结构
    project/ ├── .specify/ ✅ 保留（模板和脚本） ├── .github/ ✅ 所有copilot agent都有 ├── .claude/ ✅ 所有agent都有（包含skills/） ├── .git/ ✅ 保留 └── .vscode/ ✅ 保留
  • Skills 可用性: 所有 agent 都可以使用 .claude/skills/ 中的技能，无论安装方式

📦 部署信息

  • 内网服务器版本: 0.0.76

v0.0.75
📅 2026-04-08

🎯 本次更新重点

修复 Windows PowerShell 5.1 离线模式的关键 bug，提升跨平台兼容性和用户体验。

🐛 Bug 修复

  • 修复 PowerShell 参数冲突: 将 $Input 参数重命名为 $InputString，解决与 PowerShell 保留变量的冲突问题
    📁 .github/workflows/scripts/create-release-packages.ps1#L502
  • 修复 PowerShell 5.1 进度条崩溃: 添加全局 $ProgressPreference = 'SilentlyContinue'，解决 Compress-Archive 的 IndexOutOfRangeException 错误
    📁 .github/workflows/scripts/create-release-packages.ps1#L45
  • 修复模板子目录缺失: 在离线模式下递归复制所有模板子目录（code-review, code-review-c, skills）
    📁 src/specify_cli/__init__.py#L1353-L1354

✨ 功能改进

  • 支持 Windows PowerShell 5.1: 移除对 PowerShell 7+ 的硬性要求，Windows 内置的 PowerShell 5.1 即可使用离线模式
  • 优化离线模式体验: 添加 --script ps 参数支持，避免交互式提示阻塞后台命令执行

📚 文档更新

  • 更新首页信息: 版本号更新为 v0.0.75，移除 PS7 要求说明，添加最新更新时间显示
  • 优化安装验证: 将验证命令从 specify check 改为 specify version，输出更清晰直观
  • 新增迭代记录页面: 创建本页面，记录版本更新历史和详细变更信息

🔧 技术细节

核心问题分析

  • $Input 参数冲突: PowerShell 的 $Input 是自动变量（管道输入），使用 powershell.exe -File 调用时会导致参数值为空
  • 进度条 bug: PowerShell 5.1 的 Microsoft.PowerShell.Archive 模块在显示进度条时存在 IndexOutOfRangeException 问题，PowerShell 7+ 已修复
  • 模板目录遗漏: 原代码仅复制文件，未处理子目录结构，导致 code-review-c 和 skills 等目录内容丢失
